# Buzz Desktop Onboarding — Design Spec (Steps 1–2)

Source mockups (1280×832 each, desktop window aspect):

- `01 landing.png` — Step 1: landing / welcome
- `02a profile.png` — Step 2: profile setup (empty base state)
- `02b image modal.png` — Step 2 variant: image-picker modal open
- `02c emoji.png` — Step 2 variant: emoji picker open
- `02d filled.png` — Step 2 variant: filled-in / ready-to-continue state

---

## Theme (shared across all screens)

- **Dark theme, Catppuccin-Macchiato-like palette.** Full-bleed dark
  charcoal/navy background (≈ Macchiato `base` #24273a range), no window
  chrome in the mocks beyond the plain canvas.
- **Accent: warm yellow/gold** (Catppuccin `yellow`, ≈ #eed49f) used for the
  Buzz bee branding and for primary CTA buttons; primary buttons render with
  the yellow fill and dark text for contrast.
- **Text:** near-white primary text for headings, muted grey (`subtext`-ish)
  for supporting copy, placeholders, and helper text.
- **Surfaces:** inputs, cards, and modal panels sit on a slightly lighter
  elevated surface (`surface0`/`mantle` feel) with subtle 1px borders and
  rounded corners (~8–12px radius; the modal is larger, ~12–16px).
- Typography is a clean sans (matches the desktop app's UI font); headings
  are semibold, body/labels regular.

---

## Step 1 — Landing (`01 landing.png`)

### Layout

- Single centered column, vertically centered in the upper ~two-thirds of
  the window; the bottom ~230px of the canvas is empty background (breathing
  room). Content column is roughly 400–500px wide, horizontally centered.
- Vertical stack, center-aligned: **logo → heading → subheading → CTA
  block**, with generous spacing (~24–32px) between groups.

### Components & copy

1. **Brand mark** — the Buzz bee logo in the accent yellow, rendered large
   (~64–96px) at the top of the stack. Reads as the app icon / hexagon-bee
   motif.
2. **Heading** — large display text, near-white, semibold:
   **“Welcome to Buzz”**.
3. **Subheading** — one to two lines of muted grey text beneath the heading
   introducing the product (team chat for people + agents positioning),
   set at body size, center-aligned.
4. **Primary CTA** — full-accent yellow filled button with dark text, pill /
   rounded-rect, ~320–360px wide, the visual anchor of the screen. This is
   the “get started / create identity” action and is what advances to
   Step 2.
5. **Secondary action** — beneath the primary button, a lower-emphasis
   affordance (ghost/outline button or text link in muted grey) for the
   alternate path (returning user / existing key), visually subordinate to
   the primary CTA.

### States & navigation

- No inputs on this screen; both actions are enabled by default.
- Primary CTA → navigates to **Step 2 (profile setup, `02a`)**.
- No stepper/progress indicator is shown on the landing screen — progression
  UI begins on Step 2.

## Step 2 — Profile setup, base state (`02a profile.png`)

### Layout

- Same full-bleed dark canvas. Content is a single centered column
  (~420–480px wide), vertically centered with slightly more content than
  Step 1, running from ~y120 to ~y720.
- Stack order, center-aligned: **step/progress indicator → heading →
  supporting copy → avatar picker → display-name field → action row**.

### Components & copy

1. **Progress indicator** — small, above or near the heading: indicates
   position in the onboarding flow (step 2 of N), rendered as muted
   dots/segments with the current step highlighted in accent yellow.
2. **Heading** — semibold near-white title for the step: profile setup
   (“Set up your profile” phrasing).
3. **Supporting copy** — one line of muted grey helper text explaining that
   the name/photo is how teammates will see you.
4. **Avatar picker** — large circular avatar placeholder (~96–120px)
   centered above the form:
   - Empty state: dashed/solid subtle circle on the elevated surface color
     with a neutral placeholder glyph (person/camera) in muted grey.
   - A small circular **edit/add affordance badge** overlaps the bottom-right
     of the circle (accent-colored button with a camera/plus icon). Clicking
     the avatar (or badge) opens the **image-picker modal** (variant `02b`).
5. **Display name input** — single-line text field on elevated surface with
   subtle border, rounded corners, muted placeholder text (e.g. your name /
   display name). Label above or placeholder-only; full column width.
6. **Action row** — at the bottom of the column:
   - **Primary “Continue” button** — accent yellow style, but shown here in
     a **disabled/reduced-emphasis state** (dimmed) because the name field
     is empty.
   - **Back affordance** — lower-emphasis text/ghost button to return to the
     landing screen.

### States & navigation

- Base state = empty avatar + empty name → Continue disabled.
- Avatar click → opens `02b` modal. Emoji option inside the modal (or a
  dedicated emoji affordance) → `02c` emoji picker.
- Once a name is entered (and optionally an avatar chosen), the screen
  matches `02d` and Continue becomes enabled, advancing to Step 3.

---

## Variant `02b` — Image-picker modal (diff vs. `02a`)

- The Step-2 page remains visible behind a **dimmed scrim** (semi-opaque
  dark overlay); background content is de-emphasized but recognizable.
- A **centered modal dialog** (~450–520px wide, tall enough for its option
  list, rounded ~12–16px, elevated surface color, subtle border/shadow)
  sits over the page. Structure:
  - **Modal title** at top (choose/upload a profile photo phrasing), with a
    small **close (✕) button** in the top-right corner.
  - **Option list / body** presenting the ways to set an avatar:
    - **Upload a photo** — file-picker option, shown as a prominent row or
      drop-zone style area with an upload icon.
    - **Choose an emoji** — option that leads to the emoji-picker state
      (`02c`).
    - The options are stacked rows/cards on the modal surface with icon +
      label, hover-highlight style implied.
  - **Footer / dismiss** — cancel affordance (text button) and/or the ✕;
    dismissing returns to `02a` unchanged.
- No other page elements change; the name field and Continue state behind
  the scrim are identical to `02a`.

## Variant `02c` — Emoji picker (diff vs. `02a`)

- An **emoji-picker panel** is open, anchored to / centered near the avatar
  (popover-style panel on the elevated surface, rounded corners, shadow,
  roughly 350–450px wide and ~400–500px tall). It contains:
  - **Search field** at the top of the panel (muted placeholder “Search”).
  - **Category row** (smileys, people, nature, food, etc.) as small icon
    tabs under/over the grid.
  - **Scrollable emoji grid** — dense grid (~8–9 columns) of full-color
    emoji glyphs grouped by category with small muted section headers.
- Selecting an emoji sets it as the avatar (rendered large inside the
  circular avatar, typically on a tinted circular background) and closes
  the picker, transitioning toward the `02d` state.
- Rest of the page (heading, name field, buttons) unchanged from `02a`;
  the picker may partially overlap the form.

## Variant `02d` — Filled state (diff vs. `02a`)

- **Avatar filled:** the circular avatar now shows the chosen content — a
  selected emoji (or uploaded photo) rendered large and centered in the
  circle; the edit badge remains overlaid bottom-right so the choice can be
  changed.
- **Name entered:** the display-name field contains a real value (dark text
  on the input surface instead of muted placeholder), with the field in its
  focused/filled visual state.
- **Continue enabled:** the primary Continue button is now in its full
  accent-yellow enabled state (dark text on yellow), clearly the active
  CTA. Back affordance unchanged.
- Progress indicator, heading, and layout are otherwise identical to `02a`.
- Clicking **Continue** advances to the next onboarding step (Step 3, not
  covered by these mocks).

## Implementation notes / open questions

- Confirm exact copy strings against the design source (Figma) before
  implementation — the mockups’ small text (subtitles, placeholders,
  modal option labels) should be transcribed 1:1 rather than paraphrased.
- Accent yellow + dark canvas strongly suggests the existing Catppuccin
  Macchiato token set already used by the desktop app; reuse theme tokens
  (`yellow` accent, `base`/`surface0` surfaces, `text`/`subtext` text)
  rather than hard-coded hex values.
- Per desktop conventions: all text sizes in rem tokens (`text-base`,
  `text-sm`, `text-xs`, `text-2xs`), never px literals.
- The image modal and emoji picker should be Radix Dialog/Popover
  components to match the rest of the desktop app (mind the
  `waitForAnimations` requirement in E2E screenshots).
- Continue-button gating: disabled until display name is non-empty; avatar
  appears optional (Continue enables on name alone in `02d` semantics —
  verify with design).
