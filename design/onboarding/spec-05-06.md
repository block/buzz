## Screen 05a — Harness Selection (default state)

### Overall layout

- Full-window onboarding screen, dark theme. Solid very-dark background
  (near-black with a slight blue/indigo cast — reads as Catppuccin
  Macchiato/Mocha "base"/"crust" territory, roughly `#1e1e2e`–`#24273a`).
- Single centered column, roughly 600–640 px wide, horizontally centered in
  the 1280 px window. Generous empty margins left/right (~320 px each side).
- Vertical structure, top → bottom:
  1. Step indicator (progress dots) near the top, centered.
  2. Heading block (title + subtitle), centered text.
  3. Vertical stack of harness option cards (list-style radio cards), full
     column width.
  4. Footer navigation row pinned near the bottom of the window: "Back" on
     the left of the column, primary "Continue" button on the right.
- Alignment: everything inside cards is left-aligned; heading/subtitle and
  step dots are center-aligned; footer buttons align to the column's outer
  edges.

### Step indicator

- A horizontal row of small dots/pills centered above the heading,
  representing the onboarding steps. The current step (5th) is emphasized —
  rendered as a wider/brighter pill in the accent color (mauve/lavender
  purple), while completed/previous dots are dimmer and upcoming dots are
  faint gray. Approx. 6 total steps shown.

### Heading block

- **Title:** `Choose your harness` — large (~28–32 px), semibold/bold, near-white
  text (Catppuccin "text", ~`#cdd6f4`), centered.
- **Subtitle:** one line of muted gray text (~14–15 px, "subtext" color)
  beneath the title, explaining that the harness is the agent runtime that
  powers agents in Buzz, and that it can be changed later — copy along the
  lines of: `Pick the agent runtime that will power your agents. You can
  change this later in Settings.`
- Spacing: ~12–16 px between title and subtitle; ~32–40 px between subtitle
  and the first card.

### Harness option cards

- A vertical stack of **3 selectable cards**, each full column width
  (~600–640 px) and ~96–110 px tall, with ~12–16 px vertical gap between
  cards. Rounded corners (~10–12 px radius), 1 px subtle border
  (Catppuccin "surface0/surface1" gray), card background one step lighter
  than the page background ("mantle/surface" tone).
- Card internal layout (horizontal):
  - **Left:** a square icon/logo tile (~40–48 px, rounded) representing the
    harness product.
  - **Middle:** text block, left-aligned:
    - Harness **name** on the first line (~16 px, medium/semibold,
      near-white). One card carries a small **"Recommended"** badge/pill
      inline after its name (accent-tinted pill, small caps-ish ~11 px
      text).
    - **Description** on the second line(s) (~13–14 px, muted gray),
      one–two lines summarizing the harness.
  - **Right:** a radio-style selection indicator (empty circle outline in
    the default state), vertically centered.
- The three harnesses (top → bottom):
  1. **goose** — badged *Recommended*. Description along the lines of
     `Block's open-source AI agent framework. Works out of the box with
     Buzz.`
  2. **Claude Code** — description along the lines of `Anthropic's agentic
     coding tool. Requires a Claude subscription or API key.`
  3. **Custom (ACP)** — description along the lines of `Bring your own
     ACP-compatible agent harness. For advanced setups.`
- In 05a **no card is hovered or selected**: all cards share the same resting
  background/border, all radio circles are empty outlines.

### Footer navigation

- Bottom row within the column, ~40–56 px above the window's bottom edge:
  - **Back** — left side; ghost/tertiary style (muted gray text, no fill or a
    very subtle outline), with a small `←` affordance or plain text.
  - **Continue** — right side; primary filled button in the accent purple
    (mauve/lavender) with dark text, rounded (~8 px), ~40 px tall,
    ~120–140 px wide. In 05a (nothing selected) it appears **dimmed/disabled**
    relative to 05c.

### Colors / theme summary (05a)

| Role | Impression |
|---|---|
| Page background | Near-black dark blue-gray (Catppuccin base/crust) |
| Card background | Slightly lighter surface tone |
| Card border | Low-contrast gray (surface0/1) |
| Title text | Near-white (`text`) |
| Subtitle / descriptions | Muted gray (`subtext0/overlay`) |
| Accent (dots, badge, Continue, selection) | Mauve/lavender purple |

The palette strongly reads as **Catppuccin Macchiato/Mocha**, consistent with
Buzz's existing desktop theme.

## Screen 05b — Harness Hover (diff vs 05a)

Everything identical to 05a **except** the hover treatment on one card
(the middle/hovered card — Claude Code in the mock):

- Card **background lightens** one step (surface0 → surface1 feel).
- Card **border brightens** slightly (higher-contrast gray, not yet accent).
- Subtle affordance shift — the card reads as raised/interactive; no change
  to the radio indicator (still an empty outline circle).
- Cursor implied pointer. No change to heading, step dots, other cards, or
  footer. Continue remains in its default (disabled/dim) state.

Hover is therefore a **neutral highlight** (lighter surface + border), with
the accent color reserved for the selected state.

## Screen 05c — Harness Selected (diff vs 05a)

Identical layout/copy to 05a with the **goose (Recommended)** card selected:

- Selected card gets an **accent-colored border** (purple, ~1.5–2 px) and a
  faint accent-tinted background wash, clearly distinct from both resting and
  hover states.
- The right-side **radio indicator is filled**: accent-colored circle with an
  inner dot / check, replacing the empty outline.
- Card text stays the same color (name near-white, description muted) — the
  selection is communicated by border, tint, and radio, not text color.
- Other two cards remain in the resting state from 05a.
- **Continue** button in the footer is now **fully enabled**: solid accent
  purple fill at full opacity with dark contrasting label text — the clearest
  footer difference vs 05a.
- Step dots, heading, subtitle, Back button: unchanged.

State summary:

| State | Background | Border | Radio | Continue |
|---|---|---|---|---|
| Rest (05a) | surface | subtle gray | empty outline | dimmed/disabled |
| Hover (05b) | +1 lighter | brighter gray | empty outline | unchanged |
| Selected (05c) | accent tint | accent purple | filled accent dot | enabled, full accent |

## Screen 06 — Theme Selection

### Overall layout

- Same onboarding chrome as Step 5: dark page background, centered column
  (~640–700 px), step dots at top (now the **6th/last dot emphasized**),
  centered heading block, content area, footer nav row at the bottom.
- Content is a **horizontal row of theme preview cards** (grid of 2–3
  side-by-side cards) rather than the vertical list used in Step 5.

### Heading block

- **Title:** `Choose your theme` — same type treatment as Step 5
  (~28–32 px, semibold, near-white, centered).
- **Subtitle:** one muted line, e.g. `Pick how Buzz looks. You can change
  this anytime in Settings.`

### Theme cards

- **Three cards side by side** (equal width, ~200–210 px each with ~16–20 px
  gutters), each consisting of:
  - A **miniature app preview thumbnail** at the top of the card: a stylized
    mini mock of the Buzz chat UI (sidebar strip + message area) rendered in
    that theme's palette. Rounded corners, fills most of the card.
    - **Light** preview: pale/cream background with dark text lines —
      Catppuccin **Latte** rendering.
    - **Dark** preview: deep blue-gray background with light text lines —
      Catppuccin **Macchiato** rendering.
    - **System** preview: split/diagonal composite showing half light, half
      dark (auto-follows OS appearance).
  - A **label** beneath the preview, centered: `Light`, `Dark`, `System`.
  - A radio/selection affordance tied to the card (either a small radio
    under the label or the card border itself acting as the selector).
- **Selected state shown in the mock:** the **Dark** card is selected —
  accent purple border + filled radio, matching the Step 5 selection
  language. The other two cards show the resting subtle-gray border.
- Card chrome matches Step 5 cards: surface background, 1 px border,
  ~10–12 px radius.

### Footer navigation

- **Back** (ghost, left) and a primary accent button on the right. Because a
  theme is preselected (Dark), the primary button is **enabled**. As the
  final step its label reads as the completion action — `Continue` /
  `Finish` / `Get started` (final-step CTA; confirm exact string against the
  source file when implementing — the button is the same accent-filled
  style as 05c's enabled Continue).

### Colors / theme

- The screen itself is rendered in the dark theme (matching the selected
  "Dark" card), same Catppuccin-style palette as Step 5: near-black base,
  surface cards, near-white title, muted subtext, mauve/lavender accent for
  the active step dot, selected card border, and primary CTA.

## Cross-screen implementation notes

1. **Shared onboarding shell:** centered column, step-dot progress header,
   title + subtitle block, content slot, Back/Continue footer — identical
   across steps 5 and 6; only the content slot changes (vertical radio cards
   vs horizontal preview cards).
2. **Selection language is consistent:** neutral lighten on hover, accent
   border + accent radio on selection, accent-filled primary CTA gated on a
   selection existing.
3. **Theme tokens:** everything maps cleanly onto Catppuccin
   Latte/Macchiato tokens already used by the desktop app (`base`,
   `surface0/1`, `text`, `subtext0`, `mauve`/`lavender` accent). Use existing
   Tailwind theme tokens and rem-based text sizes (per repo text-sizing
   rules) — titles ≈ `text-2xl/3xl`, card names `text-base`, descriptions
   `text-sm`, badges `text-2xs`.
4. **Defaults:** Step 5 has no default selection (Continue disabled until a
   harness is picked; goose is merely badged *Recommended*). Step 6 arrives
   with a selection already active (Dark in the mock — likely should default
   to System in implementation; confirm with design).
